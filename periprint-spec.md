# PeriPrint — спецификация GUI-приложения для печати на Peripage

**Статус:** черновик / RFC
**Дата:** 2026-07-14
**Автор:** Vais
**Целевая платформа:** Linux (Mint/Cinnamon, HP EliteBook 840 G11), с прицелом на кросс-платформенность

---

## 1. Резюме

Десктопное GUI-приложение на `customtkinter`, которое даёт удобный UX поверх низкоуровневой библиотеки `peripage` (bitrate16/peripage-python) для печати на термопринтерах Peripage (A6/A6+/A40/A40+, включая протестированный ALD-Y200, 203dpi) через Bluetooth SPP/RFCOMM.

Проблема, которую решаем: сейчас печать — это CLI-команды с ручным подбором MAC-адреса, модели принтера, ширины, конкатенации кусков картинки из-за перегрева. Хотим drag&drop файла → предпросмотр → печать в 1-2 клика, с сохранённым профилем принтера.

**Предположения (зафиксировать/поправить при необходимости):**
- Основной принтер — Peripage A40 (ALD-Y200, 203dpi), но модель выбирается в UI, т.к. библиотека поддерживает A6/A6+/A40/A40+.
- Один пользователь, один принтер за раз (без параллельной печати на несколько устройств в MVP).
- Приоритет — Linux, т.к. это основная рабочая среда; Windows/macOS — best effort, не блокируют MVP.
- Печатаем: изображения, текст/markdown, PDF (растеризация постранично). DOCX — вне MVP (конвертация через LibreOffice headless — опционально в v2).
- Сетевая печать / несколько принтеров одновременно — вне скоупа.

## 2. Пользовательские сценарии (User Journeys)

### 2.1. Первый запуск
1. Пользователь открывает приложение.
2. Видит **минималистичный пустой экран**: только опции подключения к принтеру (крупные иконки — «Найти принтер» / статус подключения) и зона drag & drop. Никаких настроек печати, очереди или превью здесь ещё нет — см. §7.4 (прогрессивное раскрытие интерфейса).
3. Жмёт «Найти принтер» → идёт BLE/BT-скан → список найденных устройств (имя + MAC + сила сигнала, если доступна).
4. Выбирает устройство → выбирает модель (A6 / A6+ / A40 / A40+) из выпадающего списка → жмёт «Подключить и сохранить как профиль по умолчанию».
5. Профиль сохраняется в конфиг, при следующем запуске автоподключение (или быстрый коннект по кнопке).

### 2.2. Обычная печать (основной happy path)
1. Пользователь перетаскивает файл (PDF/PNG/JPG/TXT/MD) в окно приложения.
2. **Только теперь** (после приёма файла) интерфейс раскрывается: появляются очередь, превью и полный блок настроек — см. §7.4.
3. Приложение показывает превью (растрированное, как будет выглядеть на ленте — чёрно-белое, с шириной под текущий принтер).
4. Пользователь настраивает: концентрацию (яркость/температуру), настройки бумаги, качество печати, режим «целиком по формату страницы (A4/A5/Letter)» или «по длине контента» (обрезка под фактическую высоту, экономия ленты), отступы, ориентацию, обрезку/масштаб, при необходимости — конкретные страницы документа и количество копий (документа целиком или отдельных страниц) — полный список в §3 P1.
5. Жмёт «Печать» → задача уходит в очередь → прогресс-бар показывает текущий фрагмент/страницу.
6. По окончании — уведомление, запись в историю.

### 2.3. Печать нескольких файлов (batch)
1. Пользователь перетаскивает несколько файлов сразу или добавляет по одному — они попадают в очередь (список).
2. Может менять порядок (drag reorder), удалять из очереди, задавать общие настройки печати или индивидуальные на файл.
3. Жмёт «Печать всё» → последовательная печать с паузами на охлаждение между тяжёлыми заданиями (настраиваемо).

### 2.4. Восстановление после сбоя связи
1. Bluetooth-соединение обрывается в процессе печати (частый кейс с RFCOMM).
2. Приложение детектит обрыв, ставит задачу на паузу, предлагает «Переподключиться и продолжить» / «Начать заново» / «Отменить».
3. При переподключении — допечатывает с последнего успешно отправленного фрагмента (нужен трекинг прогресса по чанкам, не по документам).

### 2.5. Смена/добавление принтера
1. Пользователь открывает «Настройки → Принтеры».
2. Видит список сохранённых профилей (имя, MAC, модель, последнее использование).
3. Может добавить новый через скан, отредактировать (переименовать, сменить модель), удалить, выбрать активный.

## 3. Функциональные требования

### P0 — MVP (must have)
- [ ] Bluetooth-скан устройств, фильтрация по имени (`PeriPage*`).
- [ ] Сохранение профиля принтера (имя, MAC, модель) в локальный конфиг.
- [ ] Подключение/переподключение к принтеру с индикацией статуса (Disconnected / Connecting / Connected / Error).
- [ ] Drag & drop файлов в окно приложения (одиночный и множественный).
- [ ] Поддержка форматов: PNG/JPG/BMP, TXT, PDF (постранично).
- [ ] Растеризация документа под ширину конкретной модели принтера (384px A6 / 576px A6+ / ширина A40 по DPI).
- [ ] Превью того, что уйдёт на печать (реальный масштаб/чб-дизеринг).
- [ ] Настройки печати: концентрация (0-2), отступ/break между блоками, автонарезка длинных изображений на чанки с паузой (защита от перегрева).
- [ ] Очередь печати с прогрессом (текущий файл / чанк / % ).
- [ ] Обработка обрыва соединения: pause + retry без потери прогресса.
- [ ] Базовый лог/история печати (что, когда, каким принтером, статус успех/ошибка).

### P1 — Should have
- [ ] Markdown-рендеринг в изображение (моноширинный шрифт, заголовки крупнее) для печати заметок/чек-листов.
- [ ] Множественные профили принтеров + быстрое переключение.
- [ ] Настройка "fit to width" / "actual size" / "crop" для изображений.
- [ ] Шаблоны печати (например, "чек-лист", "QR-визитка", "этикетка") с быстрыми полями ввода.
- [ ] Печать QR-кода из текста/URL прямо из UI (без файла).
- [ ] Тёмная/светлая тема (customtkinter из коробки), но зафиксировать палитру под термо-ленту (превью всегда Ч/Б).
- [ ] Системный трей / фоновый режим (чтобы не держать окно открытым).
- [ ] **Прогрессивное раскрытие интерфейса**: при запуске/до приёма файла — только опции подключения к принтеру (крупные иконки) + drag & drop зона; остальной UI (очередь, превью, настройки) появляется только после того, как принято хотя бы одно задание. См. §7.4.
- [ ] **Настройки бумаги**: явный выбор/индикация типа и ширины установленного рулона, если отличается от ширины по умолчанию для модели принтера.
- [ ] **Настройки качества печати**: пользовательские пресеты (например "черновик" / "обычное" / "лучшее") поверх сырых параметров concentration/dithering — не только числовые слайдеры.
- [ ] **Режим печати страницы**: «целиком по формату» (A4/A5/Letter — печатать всю страницу документа, масштабируя по ширине принтера) или «по длине контента» (обрезать под фактическую высоту текста/данных без хвостовых пустых полей — экономия ленты).
- [ ] **Печать конкретных страниц документа** (диапазон, например "2-4,7") — не только весь документ целиком.
- [ ] **Печать N копий конкретных страниц.**
- [ ] **Печать N копий документа целиком.**

### P2 — Nice to have
- [ ] DOCX → PDF → печать (через `libreoffice --headless` конвертацию).
- [ ] Печать напрямую из буфера обмена (скриншот/картинка).
- [ ] Горячие клавиши / drag-drop из файлового менеджера с автопечатью по умолчанию (быстрая печать одним перетаскиванием на иконку в трее).
- [ ] Плагин-система для новых моделей принтеров (JSON-описание ширины/протокола, без правки кода).
- [ ] Статистика использования (метраж ленты, кол-во печатей, оценка расхода батареи).

### Явно вне скоупа
- Цветная печать (у Peripage её физически нет).
- Печать по Wi-Fi/сети, поддержка облачных принтеров.
- Мобильная версия — только десктоп.
- Одновременная печать на нескольких принтерах параллельно.

## 4. Архитектура

### 4.1. Слоистая структура

```
┌─────────────────────────────────────────────┐
│  UI Layer (customtkinter)                    │
│  - MainWindow, PrinterPanel, QueuePanel,      │
│    PreviewPanel, SettingsDialog               │
└───────────────┬───────────────────────────────┘
                │ вызовы через сервисный интерфейс
                │ (никогда напрямую в BT/PIL из UI-кода)
┌───────────────▼───────────────────────────────┐
│  Application / Service Layer                  │
│  - PrintJobManager (очередь, статусы)          │
│  - PrinterManager (профили, коннекшены)        │
│  - DocumentPipeline (рендер файла → растр)     │
└───────────────┬───────────────────────────────┘
                │
┌───────────────▼───────────────────────────────┐
│  Infrastructure Layer                          │
│  - BluetoothScanner (bleak/PyBluez обёртка)     │
│  - PeripageClient (обёртка над peripage-lib)    │
│  - Renderers: ImageRenderer, PdfRenderer,       │
│    TextRenderer, MarkdownRenderer               │
│  - ConfigStore (~/.config/periprint/config.json)│
│  - HistoryStore (sqlite или jsonl)              │
└─────────────────────────────────────────────────┘
```

Причина разделения: UI (customtkinter, синхронный Tk-mainloop) не должен блокироваться операциями Bluetooth (они по сути I/O-bound и нестабильны — обрывы, таймауты). Всё, что трогает BT/печать/рендеринг тяжёлых PDF, уходит в фоновые потоки, UI получает обновления через thread-safe очередь событий.

### 4.2. Threading-модель

- **Main thread**: только Tk mainloop + обработка событий из `queue.Queue` (poll каждые ~100мс через `after()`).
- **Worker thread (1 на печать)**: выполняет `PrintJobManager.run_job()` — рендер + отправка по BT, шлёт прогресс-события в очередь.
- **Scan thread**: отдельный поток на BT-скан устройств (может занимать 5-15 сек), тоже через очередь событий в UI.
- Никаких прямых вызовов Tk-виджетов из фоновых потоков (частая ошибка — краши/зависания Tkinter).

Скетч событий:
```python
ui_events: queue.Queue  # (EventType, payload)

# в воркере:
ui_events.put(("SCAN_RESULT", devices))
ui_events.put(("PRINT_PROGRESS", {"job_id": ..., "percent": 42}))
ui_events.put(("PRINT_ERROR", {"job_id": ..., "error": "...", "recoverable": True}))

# в главном потоке (через self.after(100, self._poll_events)):
def _poll_events(self):
    while not ui_events.empty():
        event, payload = ui_events.get_nowait()
        self._dispatch(event, payload)
    self.after(100, self._poll_events)
```

### 4.3. Ключевая проблема протокола: перегрев и разрыв буфера

Как обсуждали раньше — у Peripage маленький внутренний буфер (~250px по высоте) и печатающая головка перегревается на длинных/тёмных изображениях. Значит `DocumentPipeline` должен:
1. Рендерить документ в полный растр нужной ширины.
2. Резать по высоте на чанки (настраиваемый размер, дефолт ~200-250px).
3. Между чанками — управляемая пауза (`break`), с адаптивным увеличением паузы, если несколько тёмных чанков подряд (эвристика по % чёрных пикселей).
4. Каждый чанк — отдельная единица прогресса и retry (если соединение легло между чанками — допечатываем с этого чанка, не с начала документа).

## 5. Технологический стек

| Компонент | Библиотека | Комментарий |
|---|---|---|
| GUI framework | `customtkinter` | тема, кастомные виджеты поверх Tk |
| Drag & drop | `tkinterdnd2` | нативный DnD для Tk (customtkinter поверх обычного Tk, совместимо) |
| Печать (протокол) | `peripage` (bitrate16/peripage-python, PyPI: `peripage`) | поддерживает A6/A6+/A40/A40+ |
| Bluetooth-транспорт | `PyBluez` (зависимость peripage) | classic BT SPP/RFCOMM |
| BT-скан (опционально доп. слой) | `bleak` или системный `bluetoothctl`/`hcitool` через subprocess | для более стабильного скана на современных BlueZ, если PyBluez discovery глючит |
| Изображения | `Pillow` | рендер, дизеринг, ресайз |
| PDF → растр | `PyMuPDF` (`fitz`) | быстрее и без системных зависимостей, чем `pdf2image`/poppler |
| Markdown → изображение | `Pillow.ImageDraw` + простой markdown-парсер (`markdown` + `PIL`) или готовый рендер через `weasyprint` (тяжелее) | для MVP хватит своего простого рендера моноширинным шрифтом |
| Конфиг | `json` в `~/.config/periprint/config.json` | простой формат, без лишних зависимостей |
| История печати | `sqlite3` (stdlib) | легче джойнить/фильтровать чем jsonl, но jsonl проще для MVP |
| Упаковка (опционально) | `pyinstaller` | если понадобится standalone-бинарник |

Python 3.11+ (соответствует текущему окружению на EliteBook).

## 6. Модель данных

```python
from dataclasses import dataclass, field
from enum import Enum
from datetime import datetime

class PrinterModel(str, Enum):
    A6 = "A6"
    A6_PLUS = "A6p"
    A40 = "A40"
    A40_PLUS = "A40p"

@dataclass
class PrinterProfile:
    id: str                 # uuid
    name: str                # "Мой A40"
    mac: str                 # "00:15:83:15:bc:5f"
    model: PrinterModel
    default_concentration: int = 1     # 0-2
    default_break: int = 60            # px
    chunk_height_px: int = 220         # защита от перегрева
    last_connected_at: datetime | None = None

class DocumentKind(str, Enum):
    IMAGE = "image"
    TEXT = "text"
    MARKDOWN = "markdown"
    PDF = "pdf"

class JobStatus(str, Enum):
    QUEUED = "queued"
    RENDERING = "rendering"
    PRINTING = "printing"
    PAUSED_ERROR = "paused_error"
    DONE = "done"
    FAILED = "failed"
    CANCELLED = "cancelled"

@dataclass
class PrintSettings:
    concentration: int = 1
    break_px: int = 60
    fit_mode: str = "fit_width"   # fit_width | actual_size | crop
    dithering: bool = True
    margin_top_px: int = 0
    margin_bottom_px: int = 40    # запас на отрыв ленты

@dataclass
class DocumentItem:
    id: str
    source_path: str
    kind: DocumentKind
    settings: PrintSettings = field(default_factory=PrintSettings)

@dataclass
class PrintJob:
    id: str
    document: DocumentItem
    printer_profile_id: str
    status: JobStatus = JobStatus.QUEUED
    total_chunks: int = 0
    completed_chunks: int = 0
    error_message: str | None = None
    created_at: datetime = field(default_factory=datetime.now)
```

### 7.2. Ключевые экраны/диалоги
1. **Главное окно** — как выше: очередь слева, превью+настройки справа, статус-бар снизу.
2. **Выбор принтера / скан** — модалка со списком найденных устройств, кнопка "Сканировать ещё раз", ручной ввод MAC (fallback, если скан не находит).
3. **Настройки → Принтеры** — CRUD над сохранёнными профилями.
4. **Настройки → Приложение** — путь конфига, тема, размер чанка по умолчанию, автоподключение при старте.
5. **История** — таблица (дата, файл, принтер, статус, длительность), с возможностью повторной печати одним кликом ("Печать снова").
6. **Диалог ошибки соединения** — "Не удалось отправить чанк 4/9. [Переподключиться] [Отменить] [Пропустить чанк]".

### 7.3. Принципы UX
- Всегда показывать реальный чёрно-белый превью — то, что человек видит в UI, должно максимально соответствовать физической печати (дизеринг Флойда-Стейнберга для полутонов, т.к. термопринтер бинарный — либо точка есть, либо нет).
- Прогресс — по чанкам, не по файлам целиком: пользователь должен видеть, что происходит на низком уровне (полезно для диагностики зависаний BT).
- Не блокировать UI никогда — статус "Connecting..." со спиннером, не заморозка окна.
- Явно показывать состояние принтера (батарея, если библиотека это умеет через `-e`/introduce) в шапке — полезно, чтобы не начать печать на разряженном принтере.

### 7.4. Прогрессивное раскрытие интерфейса и визуальный дизайн

**Дизайн-направление:** визуально ориентироваться на принципы из
[design.md](https://getdesign.md/composio/design-md) (см. также обзор подходов на
[awesome-design-md](https://github.com/voltagent/awesome-design-md)) — но это про визуальный
язык (отступы, иерархия, типографика, использование иконок), а не про смену технологии:
приложение остаётся на `customtkinter`/Tkinter, никакого перехода на веб-стек/другой UI
framework.

**Два состояния главного окна** (не одно статичное, как в §7.1):

1. **Пустое состояние (до приёма файла).** Видны только:
   - Опции подключения к принтеру — крупными иконками (не мелкий текст/выпадающий список),
     сразу считывается статус подключения.
   - Зона drag & drop, занимающая большую часть окна.

   Никакой очереди, превью или блока настроек печати на этом экране нет — они не просто
   пустые/задизейбленные, а **не отрендерены вообще**, чтобы не перегружать первый экран.

   ```
   ┌────────────────────────────────────────────────────┐
   │  PeriPrint                                    [⚙]   │
   │                                                      │
   │              [ 🖨️ ]      [ 🔍 ]                      │
   │           Мой A40      Найти принтер                │
   │           ● Connected                                │
   │                                                      │
   │     ┌──────────────────────────────────────────┐    │
   │     │                                          │    │
   │     │     Перетащите файлы сюда                 │    │
   │     │     или нажмите для выбора                │    │
   │     │                                          │    │
   │     └──────────────────────────────────────────┘    │
   │                                                      │
   └────────────────────────────────────────────────────┘
   ```

2. **Раскрытое состояние (после приёма ≥1 файла).** Интерфейс разворачивается в layout из
   §7.1 (очередь + превью + настройки), дополненный новыми блоками настроек из §3 P1:
   настройки бумаги, качества, режима «целиком по формату» / «по длине контента», выбор
   конкретных страниц и количества копий (страниц или документа целиком).

Переход между состояниями — при приёме первого файла (drag&drop или выбор через диалог);
обратный переход (в пустое состояние) — когда очередь становится полностью пустой (все
задания напечатаны и очищены).

## 8. Пайплайн обработки документа

```
Файл → определение типа (по расширению/MIME)
     → Renderer (Image/PDF/Text/Markdown) → PIL.Image (RGB/L), ширина = printer.native_width_px
     → нормализация в 1-bit (дизеринг или порог) под термопечать
     → нарезка на чанки по chunk_height_px
     → PrintJobManager кладёт чанки в очередь отправки
     → PeripageClient.send_chunk(chunk_bytes) через peripage-lib
     → на успех — commit прогресса job; на ошибку — pause job, событие в UI
```

`PeripageClient` — тонкая обёртка над `peripage.Printer`, добавляющая:
- Явный `connect()`/`disconnect()`/`is_connected()` с таймаутами.
- Реконнект с экспоненциальным backoff (макс. 3 попытки, потом — ручное решение пользователя).
- Учёт `printer.model.native_width_px` (константа из библиотеки на модель — 384 для A6, 576 для A6+, ширина по DPI для A40).

## 10. Обработка ошибок и edge cases

| Ситуация | Поведение |
|---|---|
| BT выключен в системе | Баннер "Включите Bluetooth", кнопка ретрая скана |
| Принтер не отвечает при подключении | Таймаут 10с → статус Error, предложить повторить/выбрать другой профиль |
| Обрыв соединения в середине чанка | Job → PAUSED_ERROR, чанк не считается завершённым, ретрай с того же чанка |
| Слишком большое изображение (по памяти) | Предупреждение + авто-downscale с подтверждением пользователя |
| Неподдерживаемый формат файла | Toast "Формат .xyz не поддерживается", файл не добавляется в очередь |
| PDF с паролем/битый PDF | Ошибка рендера → файл помечается Failed с текстом причины, остальная очередь продолжает работу |
| Принтер разрядился в процессе (если библиотека умеет читать battery) | Предупреждение при <15%, пауза очереди с уведомлением |
| Несколько экземпляров приложения запущены одновременно | Лок-файл в конфиг-директории, второй экземпляр либо фокусирует первый, либо предупреждает |

## 11. Нефункциональные требования

- **Платформа:** Linux Mint 22.x/Cinnamon 6.6.x — основная целевая среда, тестируем в первую очередь тут. Windows/macOS — не ломать сборку, но не тратим время на специфичные баги вне MVP.
- **Производительность:** рендер + печать одной A4-страницы PDF (203dpi) — рендер должен занимать <2с на EliteBook 840 G11; сама печать ограничена скоростью принтера (не оптимизируем).
- **Надёжность:** ни один сбой BT не должен приводить к падению приложения — все BT-вызовы обёрнуты в try/except с логированием и явным UI-статусом.
- **Память:** не держать в памяти растры всех файлов очереди одновременно — рендерить лениво (по мере того, как job берётся в работу), либо кэшировать превью в уменьшенном виде.

## 12. План разработки (этапы)

### Этап 0 — Прототип протокола (0.5 дня)
- Скрипт на голом `peripage` CLI/lib, проверка печати текста/картинки на реальном ALD-Y200, замер `native_width_px` и стабильного `chunk_height_px` эмпирически.

### Этап 1 — MVP скелет (2-3 дня)
- Каркас `customtkinter`-приложения: главное окно, статичная разметка (без реальной логики).
- `PrinterManager` + сохранение/загрузка профилей (без реального BT — заглушка).
- `ConfigStore`.

### Этап 2 — Bluetooth-интеграция (2 дня)
- Реальный скан устройств (поток + событийная очередь в UI).
- Подключение/отключение, обработка ошибок, retry.

### Этап 3 — Document pipeline (2-3 дня)
- Renderers: Image, PDF (PyMuPDF), Text.
- Нарезка на чанки, дизеринг, генерация превью.

### Этап 4 — Печать + очередь (2 дня)
- `PrintJobManager`, воркер-поток, прогресс по чанкам.
- Реконнект/резюм печати.

### Этап 5 — Полировка UX (1-2 дня)
- Drag&drop, темы, история, статус-бар, диалоги ошибок.

### Этап 6 (P1, отдельно) — Markdown-рендер, шаблоны, QR из UI, трей.

Итого MVP (этапы 0-5): ~9-12 дней в темпе "вечера/выходные"; в фултайм-режиме — 3-4 дня.

## 13. Тестирование

- **Unit:** рендереры (Image/PDF/Text → корректный размер/формат растра), нарезка на чанки (граничные случаи: высота ровно кратна/не кратна chunk_height).
- **Интеграционные (моки BT):** `PeripageClient` с фейковым транспортом — проверка retry-логики, восстановления с середины job.
- **Ручные (на реальном принтере):** чек-лист — печать текста, картинки, многостраничного PDF, обрыв соединения вручную (выключить BT посреди печати) → проверить резюм.

## Приложение: справка по низкоуровневому протоколу (для реализации PeripageClient)

- Транспорт: Bluetooth Classic, профиль SPP, RFCOMM-канал.
- Библиотека `peripage` (PyPI) — поддерживаемые модели: `A6`, `A6p`, `A40`, `A40p`.
- CLI для референса при отладке:
  ```bash
  peripage -m <MAC> -p A40 -c 1 -b 60 -i image.png
  ```
- Известные ограничения библиотеки: внутренний буфер принтера ~250px по высоте, перегрев при длинной/тёмной печати без пауз — учтено в `chunk_height_px` и адаптивных паузах (см. §4.3).
- На некоторых системах современный BlueZ конфликтует с discovery через PyBluez — предусмотреть fallback на `bluetoothctl`/`hcitool` через subprocess, если библиотечный скан не находит устройства.
